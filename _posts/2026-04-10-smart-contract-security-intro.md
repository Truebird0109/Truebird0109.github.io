## 왜 스마트 컨트랙트가 해킹당하나?

일반 웹사이트랑 근본적으로 다른 점이 있다.

| | 웹사이트 | 스마트 컨트랙트 |
|---|---|---|
| 코드 공개 | 서버 코드 비공개 | **모든 코드 공개** |
| 수정 | 패치 배포하면 끝 | **수정 불가** |
| 돈 | DB에 숫자 | **코드 안에 진짜 돈** |
| 실행 환경 | 내 서버 | **전 세계 공개 네트워크** |

코드가 공개되어 있고, 안에 진짜 돈이 있고, 고칠 수 없다.

**해커에게 이보다 좋은 환경이 있을까?**

## 공격 표면 맵

```
                    ┌─────────────────────┐
                    │   Frontend (Web)     │ ← 피싱, 악성 approve 유도
                    └──────────┬──────────┘
                               │
                    ┌──────────▼──────────┐
                    │   RPC / Node        │ ← MEV, Front-running
                    └──────────┬──────────┘
                               │
┌──────────────┐   ┌──────────▼──────────┐   ┌──────────────┐
│ External     │◄──│  Smart Contract     │──►│ Oracle       │
│ Contracts    │   │  (공격 대상)         │   │ (가격 데이터) │
└──────────────┘   └──────────┬──────────┘   └──────────────┘
                               │
                    ┌──────────▼──────────┐
                    │   Blockchain State   │ ← 상태 조작
                    └─────────────────────┘
```

공격 벡터가 사방에 있다. 컨트랙트 자체 취약점, 외부 의존성, 프론트엔드, 인프라 전부.

## 대표 취약점 카테고리

### Tier 1: 자주 터지고 피해가 큰 것들

| 취약점 | 한 줄 설명 | 다루는 편 |
|--------|-----------|-----------|
| **Reentrancy** | 함수 실행 중에 다시 호출해서 돈 빼먹기 | #11 |
| **Access Control** | 관리자 전용 함수를 아무나 호출 | #12 |
| **Flash Loan Attack** | 무담보 대출로 시장 조작 | #13 |
| **Oracle Manipulation** | 가격 데이터 조작 | #14 |

### Tier 2: 알아둬야 하는 것들

| 취약점 | 한 줄 설명 |
|--------|-----------|
| **Integer Overflow/Underflow** | 숫자가 넘쳐서 엉뚱한 값 (0.8.0 이후 자동 방지) |
| **Front-running / MEV** | 내 트랜잭션을 미리 보고 앞에 끼어들기 |
| **Denial of Service** | 컨트랙트를 먹통으로 만들기 |
| **Signature Replay** | 한 번 쓴 서명을 다시 사용 |
| **Delegatecall Injection** | 남의 코드를 내 컨텍스트에서 실행 |

## 보안 감사(Audit)는 뭐 하는 거야?

스마트 컨트랙트를 배포하기 전에 **전문가가 코드를 리뷰**하는 것.

```
프로젝트 → 감사 업체에 코드 제출
  ↓
감사자: 코드 라인 하나하나 읽으며 취약점 탐색 (1~4주)
  ↓
감사 보고서 발행 (Critical / High / Medium / Low / Info)
  ↓
프로젝트: 취약점 수정
  ↓
배포
```

대표적인 감사 업체:
- **Trail of Bits** — 최상급. 해킹 대회 우승 출신들
- **OpenZeppelin** — 라이브러리도 만들고 감사도 함
- **Consensys Diligence** — 이더리움 생태계 핵심
- **Spearbit** — 독립 감사자 네트워크

감사 비용? 작은 프로젝트도 **수천만 원**, 큰 프로토콜은 **수억 원**.

## 보안 도구들

### 정적 분석 (코드만 보고 취약점 찾기)

- **Slither** — Trail of Bits가 만듦. 가장 널리 쓰이는 도구
- **Mythril** — EVM 바이트코드 분석
- **Semgrep** — 패턴 매칭 기반 (Web2에서도 유명)

### 퍼징 (랜덤 입력으로 버그 찾기)

- **Echidna** — 솔리디티 전용 퍼저
- **Foundry (forge fuzz)** — 요즘 대세

### 형식 검증 (수학적으로 증명)

- **Certora** — "이 함수는 절대 이런 상태가 될 수 없다"를 수학적으로 증명
- **Halmos** — 심볼릭 실행 기반

---

## 보안 마인드셋

코드를 읽을 때 항상 이 질문을 던져라:

1. **"이 함수를 아무나 호출하면 어떻게 되지?"** → Access Control
2. **"외부 호출 후에 상태를 변경하면?"** → Reentrancy
3. **"이 숫자가 0이면? 최대값이면?"** → Edge Case
4. **"이 가격 데이터는 누가 제공하지?"** → Oracle 의존성
5. **"한 트랜잭션 안에서 이걸 반복하면?"** → Flash Loan 가능성

이 5개 질문만 습관화해도 **상당수의 취약점**을 발견할 수 있다.

---

## 정리

- 스마트 컨트랙트: 코드 공개 + 돈 보유 + 수정 불가 = 해커의 천국
- 대표 취약점: Reentrancy, Access Control, Flash Loan, Oracle Manipulation
- 보안 감사 = 배포 전 전문가 리뷰 (비쌈)
- Slither, Echidna, Foundry 등 도구 활용
- **5가지 질문**을 습관화하자
