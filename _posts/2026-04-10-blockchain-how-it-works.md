## 블록이란?

블록체인의 **블록(Block)**은 거래 기록의 묶음입니다.

```
┌──────────────────────────┐
│ Block #100               │
├──────────────────────────┤
│ 이전 블록 해시: 0xabc... │
│ 타임스탬프: 2026-04-10   │
│ 거래 목록:               │
│   A → B: 1 ETH           │
│   C → D: 0.5 ETH         │
│ 이 블록 해시: 0xdef...   │
└──────────────────────────┘
```

핵심은 **"이전 블록 해시"**가 포함된다는 점입니다. 이것이 체인을 형성합니다.

---

## 해시(Hash)란?

어떤 데이터든 입력하면 **고정된 길이의 값**이 출력되는 함수입니다.

```
"hello" → 0x2cf24dba...
"hello!" → 0xce06092f...   ← 한 글자만 바꿔도 완전히 달라집니다
```

해시의 특성:
- **단방향**: 결과값에서 원본을 역추적할 수 없습니다
- **결정적**: 같은 입력 → 항상 같은 출력
- **충돌 저항**: 다른 입력이 같은 출력을 내기가 사실상 불가능
- **눈사태 효과**: 입력이 조금만 바뀌어도 출력이 완전히 달라짐

---

## 왜 조작이 불가능한가?

Block #100을 조작한다고 가정해보겠습니다.

1. 거래 내용을 변경하면 → Block #100의 **해시가 달라집니다**
2. Block #101에는 "이전 블록 해시 = Block #100의 원래 해시"가 기록되어 있습니다
3. 해시가 일치하지 않으니 → Block #101도 수정해야 합니다
4. Block #101을 수정하면 → Block #102도... #103도...

**체인 전체를 다시 계산해야 합니다.** 그리고 그 속도가 정직하게 운영되는 네트워크보다 빨라야 합니다. 사실상 불가능한 일입니다.

```
[Block 98] ← [Block 99] ← [Block 100 🔧 조작!] ← [Block 101 ❌ 해시 불일치!]
                                                  ← [Block 102 ❌]
                                                  ← [Block 103 ❌]
                                                  ...전체를 다시 계산해야 합니다
```

## 합의 알고리즘 — 블록을 누가 만드나요?

블록을 아무나 만들 수 없습니다. 누가 만들지를 결정하는 규칙이 **합의 알고리즘**입니다.

### PoW (Proof of Work) — 작업 증명

비트코인이 사용하는 방식입니다.

> "이 수학 문제를 가장 먼저 푼 노드가 블록을 생성합니다"

- 막대한 컴퓨팅 파워(전기)를 소모합니다
- 이 과정을 **채굴(mining)**이라고 부릅니다
- 51% 공격: 네트워크 해시파워의 절반 이상을 장악하면 조작이 가능하지만, 현실적으로는 거의 불가능합니다

### PoS (Proof of Stake) — 지분 증명

이더리움이 2022년에 전환한 방식입니다.

> "ETH를 많이 예치한 검증자가 블록을 생성합니다"

- 전력 소모가 대폭 줄었습니다
- **32 ETH**를 스테이킹하면 검증자(Validator) 자격을 얻습니다
- 악의적인 행동을 하면 예치한 ETH가 **슬래싱(삭감)**됩니다

---

## 보안 관점에서 중요한 포인트

| 개념 | 왜 중요한가? |
|------|-------------|
| 해시 | 블록체인 무결성의 근간. 해시 충돌이 발생하면 체인이 무너집니다 |
| 합의 알고리즘 | 51% 공격, Nothing-at-Stake 등 프로토콜 레벨 공격의 출발점 |
| 불변성 | 스마트 컨트랙트 배포 후 **수정 불가** → 버그가 있으면 영구히 존재합니다 |

마지막 항목이 핵심입니다. 웹사이트는 버그를 발견하면 패치를 배포하면 됩니다. **스마트 컨트랙트는 한번 올리면 그것으로 끝입니다.** 그래서 Web3 보안이 중요한 이유입니다.

---

## 정리

- 블록 = 거래 묶음 + 이전 블록 해시
- 해시가 체인을 연결하고, 조작을 방지합니다
- PoW(채굴) vs PoS(스테이킹)으로 블록 생성자를 결정합니다
- 블록체인은 불변 → 스마트 컨트랙트 버그는 **영구적**입니다
