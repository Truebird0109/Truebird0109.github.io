## 블록이 뭔데

블록체인의 **블록(Block)**은 거래 기록 묶음이다.

```
┌──────────────────────────┐
│ Block #100               │
├──────────────────────────┤
│ 이전 블록 해시: 0xabc... │
│ 타임스탬프: 2026-04-10   │
│ 거래 목록:               │
│   A → B: 1 ETH           │
│   C → D: 0.5 ETH         │
│ 이 블록 해시: 0xdef...   │
└──────────────────────────┘
```

핵심은 **"이전 블록 해시"**가 들어있다는 것이다. 이게 체인을 만든다.

---

## 해시(Hash)란?

어떤 데이터든 넣으면 **고정된 길이의 값**이 나오는 함수다.

```
"hello" → 0x2cf24dba...
"hello!" → 0xce06092f...   ← 한 글자만 바꿔도 완전히 다름
```

해시의 특성:
- **단방향**: 결과값에서 원본을 역추적 못 함
- **결정적**: 같은 입력 → 항상 같은 출력
- **충돌 저항**: 다른 입력이 같은 출력을 내기 거의 불가능
- **눈사태 효과**: 입력이 살짝만 바뀌어도 출력이 완전히 달라짐

---

## 왜 조작이 안 되나?

Block #100을 조작한다고 치자.

1. 거래 내용을 바꾸면 → Block #100의 **해시가 달라짐**
2. Block #101에는 "이전 블록 해시 = Block #100의 원래 해시"가 적혀있음
3. 해시가 안 맞으니 → Block #101도 수정해야 함
4. Block #101을 수정하면 → Block #102도... → #103도...

**체인 전체를 다시 계산해야 한다.** 그리고 그 속도가 정직한 네트워크보다 빨라야 한다. 사실상 불가능.

```
[Block 98] ← [Block 99] ← [Block 100 🔧 조작!] ← [Block 101 ❌ 해시 불일치!]
                                                  ← [Block 102 ❌]
                                                  ← [Block 103 ❌]
                                                  ...전부 다시 해야 함
```

---

## 합의 알고리즘 — 누가 블록을 만드나?

블록을 아무나 만들면 안 된다. 누가 만들지 정하는 규칙이 **합의 알고리즘**.

### PoW (Proof of Work) — 작업 증명

비트코인이 쓰는 방식.

> "이 수학 문제를 제일 먼저 푼 놈이 블록을 만든다"

- 엄청난 컴퓨팅 파워(전기) 소모
- 그래서 **채굴(mining)**이라 부름
- 51% 공격: 네트워크 해시파워의 절반 이상을 장악하면 조작 가능 (현실적으로 거의 불가)

### PoS (Proof of Stake) — 지분 증명

이더리움이 2022년에 전환한 방식.

> "돈(ETH)을 많이 맡긴 놈이 블록을 만든다"

- 전기 거의 안 씀
- **32 ETH** 스테이킹하면 검증자(Validator) 자격
- 나쁜 짓 하면? 맡긴 ETH가 **슬래싱(삭감)** 됨

---

## 보안 관점에서 중요한 포인트

| 개념 | 왜 중요? |
|------|----------|
| 해시 | 블록체인 무결성의 근간. 해시 충돌이 발생하면 체인이 깨짐 |
| 합의 알고리즘 | 51% 공격, Nothing-at-Stake 등 프로토콜 레벨 공격의 출발점 |
| 불변성 | 스마트 컨트랙트 배포 후 **수정 불가** → 버그가 있으면 영원히 존재 |

마지막 줄이 핵심이다. 웹사이트는 버그 발견하면 패치하면 된다. **스마트 컨트랙트는 한번 올리면 끝이다.** 그래서 Web3 보안이 중요한 거다.

---

## 정리

- 블록 = 거래 묶음 + 이전 블록 해시
- 해시가 체인을 잇고, 조작을 방지
- PoW(채굴) vs PoS(스테이킹)로 블록 생성자 결정
- 블록체인은 불변 → 스마트 컨트랙트 버그는 **영구적**
