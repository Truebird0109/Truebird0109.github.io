## Web3 버그 바운티 플랫폼

### Immunefi — 1순위

Web3 버그 바운티의 **대표 플랫폼**입니다.

- Web3 전문 플랫폼
- Critical 보상: **$10,000 ~ $10,000,000**
- 누적 지급 보상: **$1억 이상**
- 대부분의 주요 DeFi 프로토콜이 프로그램을 운영합니다

**시작 방법:**
1. 프로그램 목록에서 관심 있는 프로젝트 선택
2. Scope(범위) 확인 — 어떤 컨트랙트가 대상인지 확인
3. 코드 분석
4. 취약점 발견 시 보고서 작성
5. 트리아지 → 검증 → 보상

### Code4rena — 경쟁형 감사

- **감사 대회** 방식. 정해진 기간 안에 여러 참가자가 취약점을 찾습니다
- 기간: 보통 3~7일
- 상금 풀: $20,000 ~ $200,000+
- **입문자에게 적합합니다** — 대회 종료 후 다른 참가자의 보고서를 보며 학습 가능
- 대회 종료 후 **모든 발견 사항이 공개**됩니다

### Sherlock

- Code4rena와 유사한 경쟁형 감사
- 고정 보상 + 순위 기반 보상
- 감사 품질에 대한 **보험** 제공

### Hats Finance

- 탈중앙화 버그 바운티 프로토콜
- 온체인에서 보상을 관리합니다

## 보상 등급

| 등급 | 설명 | 일반적 보상 |
|------|------|------------|
| **Critical** | 자금 직접 탈취 가능 | $50,000 ~ $10,000,000 |
| **High** | 자금 손실 가능성 높음 | $10,000 ~ $50,000 |
| **Medium** | 제한적 영향 | $1,000 ~ $10,000 |
| **Low** | 낮은 영향, 비효율 등 | $100 ~ $1,000 |

Critical 취약점 하나가 **연봉 수준의 보상**으로 이어질 수 있습니다.

### Phase 2: 실전 감각 키우기 (2~4개월)

```
□ Code4rena 과거 대회 보고서 읽기 (매일 1~2개)
```

**과거 보고서 읽기가 가장 효율적인 학습 방법입니다.** 실제로 자금이 걸렸던 취약점이 어떻게 발견되었는지 살펴보는 것이 중요합니다.

### Phase 3: 독립 헌터 (4개월~)

```
□ Immunefi에서 프로그램 선택 후 본격 헌팅

□ 특정 분야 전문화 (렌딩, DEX, 브릿지 등)
□ 커뮤니티 참여 (Discord, Twitter/X)

```

## 보고서 작성법

우수한 보고서의 구조:

```markdown
## 제목
취약점을 간결하게 설명

## 심각도
Critical / High / Medium / Low

## 요약
1~2문장으로 핵심 내용 설명

## 취약점 상세
- 어떤 컨트랙트의 어떤 함수가 문제인지
- 왜 문제인지 (코드 레벨)
- 전제 조건이 있다면 명시

## 공격 시나리오
1. 공격자가 X를 수행합니다
2. Y가 발생합니다
3. 결과: Z만큼의 자금 탈취 가능

## PoC (Proof of Concept)
Foundry 테스트 코드로 실제 익스플로잇 증명

## 영향
최대 피해 금액, 영향 범위

## 수정 권장사항
구체적인 코드 수정 방안
```

**PoC 없이는 보상 확률이 크게 낮아집니다.** Foundry로 테스트 코드를 작성하는 능력이 필수입니다.

## 보안 마인드셋

### 1. 프로토콜을 먼저 이해하세요

코드를 바로 읽지 말고, **문서(docs)를 먼저 읽으세요.** 프로토콜이 무엇을 하려는 것인지 이해해야 "이것이 의도된 동작인가, 버그인가"를 판단할 수 있습니다.

### 2. 경계값을 검토하세요

- 잔고가 0이라면? 최대값이라면?
- 첫 번째 사용자라면? 마지막 사용자라면?
- 풀에 유동성이 없다면?
- 토큰 decimals가 6이라면? 18이라면?

### 3. 조합을 생각하세요

단일 함수는 안전해도, **여러 함수를 특정 순서로 호출하면** 취약점이 나타나는 경우가 많습니다.

### 4. 꾸준히 지속하세요

첫 유효 보고서까지 **수개월**이 걸릴 수 있습니다. 이것은 정상적인 과정입니다.

> "모든 보안 연구자의 첫 번째 발견 이전에는 수백 시간의 빈손이 있었습니다."
