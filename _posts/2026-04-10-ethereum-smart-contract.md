## 비트코인 vs 이더리움

비트코인 = **디지털 금**. 보내고 받는 것이 전부입니다.

이더리움 = **디지털 컴퓨터**. 돈 위에 **프로그램**을 올릴 수 있습니다.

이 프로그램이 바로 **스마트 컨트랙트(Smart Contract)**입니다.

---

## 스마트 컨트랙트란?

> 블록체인 위에서 동작하는 자동 실행 프로그램

자판기를 생각하면 이해가 쉽습니다:
1. 돈을 넣습니다 (트랜잭션 전송)
2. 버튼을 누릅니다 (함수 호출)
3. 음료가 나옵니다 (실행 결과)

**중간에 사람이 개입하지 않습니다.** 코드에 정의된 대로만 실행됩니다.

### 현실과의 비교

**전통적인 계약:**
> A와 B가 내기를 합니다. "내일 비가 오면 A가 이기고, 안 오면 B가 이긴다."
> → 결과가 나와도 진 쪽이 이행을 거부할 수 있습니다. **신뢰 문제입니다.**

**스마트 컨트랙트:**
> 양쪽 모두 돈을 컨트랙트에 예치합니다.
> → 내일 날씨 데이터가 입력되면 **자동으로** 이긴 쪽에 송금됩니다.
> → 어느 쪽도 결과를 거부할 수 없습니다.

---

## EVM — 이더리움 가상 머신

스마트 컨트랙트가 실행되는 환경이 **EVM(Ethereum Virtual Machine)**입니다.

```
개발자가 솔리디티(Solidity) 코드 작성
        ↓
컴파일러가 바이트코드로 변환
        ↓
바이트코드가 블록체인에 배포
        ↓
EVM이 바이트코드를 실행
```

EVM의 특성:
- **튜링 완전**: 이론적으로 어떤 프로그램이든 실행 가능합니다
- **결정적**: 같은 입력 → 전 세계 모든 노드에서 동일한 결과
- **격리**: 컨트랙트끼리 직접 메모리를 공유하지 않습니다
- **가스 제한**: 무한루프를 방지합니다 (가스가 소진되면 실행이 멈춥니다)

---

## 컨트랙트의 생애주기

```
1. 작성 (Solidity로 코딩)
2. 컴파일 (바이트코드 생성)
3. 배포 (트랜잭션으로 블록체인에 올림)
4. 실행 (누구든 함수 호출 가능)
5. 영구 존재 (삭제 불가... 원칙적으로)
```

5번이 핵심입니다. **배포 후에는 코드를 수정할 수 없습니다.**

버그가 있어도 그대로입니다.
취약점이 있어도 그대로입니다.

그래서 **배포 전 보안 감사(Audit)**가 생사를 가를 수 있습니다.

---

## 보안 관점에서 기억할 것

1. **코드가 곧 법입니다** — 코드에 허점이 있으면 공격자가 이를 이용해 자금을 인출할 수 있습니다
2. **모든 것이 공개됩니다** — 컨트랙트 코드, 상태, 트랜잭션 전부를 누구나 볼 수 있습니다
3. **수정 불가** — 취약점을 발견해도 패치가 불가능합니다 (업그레이드 패턴이 있지만, 그것도 복잡합니다)
4. **조합의 위험** — 컨트랙트끼리 서로 호출이 가능하므로 예상치 못한 상호작용이 발생할 수 있습니다

---

## 정리

- 이더리움 = 블록체인 + 프로그래밍
- 스마트 컨트랙트 = 블록체인 위의 자동 실행 프로그램
- EVM이 컨트랙트를 실행하며, 가스로 비용을 제한합니다
- 배포 후 수정 불가 → 보안이 생명입니다
