## 비트코인 vs 이더리움

비트코인 = **디지털 금**. 보내고 받는 게 끝이다.

이더리움 = **디지털 컴퓨터**. 돈 위에 **프로그램**을 올릴 수 있다.

이 프로그램이 바로 **스마트 컨트랙트(Smart Contract)**다.

---

## 스마트 컨트랙트란?

> 블록체인 위에서 돌아가는 자동 실행 프로그램

자판기를 생각하면 된다:
1. 돈 넣기 (트랜잭션 보내기)
2. 버튼 누르기 (함수 호출)
3. 음료 나옴 (실행 결과)

**중간에 사람이 개입하지 않는다.** 코드대로만 실행된다.

### 현실 예시로 비교

**전통적인 계약:**
> A와 B가 내기를 한다. "내일 비 오면 A가 이기고, 안 오면 B가 이긴다."
> → 결과가 나와도 진 쪽이 안 줄 수 있음. **신뢰 문제.**

**스마트 컨트랙트:**
> 양쪽 다 돈을 컨트랙트에 넣어둔다.
> → 내일 날씨 데이터가 들어오면 **자동으로** 이긴 쪽에 송금.
> → 누구도 결과를 거부할 수 없음.

---

## EVM — 이더리움 가상 머신

스마트 컨트랙트가 실행되는 환경이 **EVM(Ethereum Virtual Machine)**이다.

```
개발자가 솔리디티(Solidity) 코드 작성
        ↓
컴파일러가 바이트코드로 변환
        ↓
바이트코드가 블록체인에 배포
        ↓
EVM이 바이트코드를 실행
```

EVM의 특징:
- **튜링 완전**: 이론적으로 어떤 프로그램이든 실행 가능
- **결정적**: 같은 입력 → 전 세계 모든 노드에서 같은 결과
- **격리**: 컨트랙트끼리 직접 메모리 공유 안 함
- **가스 제한**: 무한루프 방지 (돈이 다 떨어지면 멈춤)

---

## 컨트랙트의 생애주기

```
1. 작성 (Solidity로 코딩)
2. 컴파일 (바이트코드 생성)
3. 배포 (트랜잭션으로 블록체인에 올림)
4. 실행 (누구든 함수 호출 가능)
5. 영원히 존재 (삭제 불가... 원칙적으로)
```

5번이 중요하다. **배포하면 코드를 수정할 수 없다.**

버그가 있어도? 그대로다.
취약점이 있어도? 그대로다.

그래서 **배포 전 보안 감사(Audit)**가 생사를 가른다.

---

## 보안 관점에서 기억할 것

1. **코드가 곧 법이다** — 코드에 구멍이 있으면 해커가 합법적(?)으로 돈을 빼감
2. **모든 것이 공개다** — 컨트랙트 코드, 상태, 트랜잭션 전부 누구나 볼 수 있음
3. **수정 불가** — 취약점 발견해도 패치가 안 됨 (업그레이드 패턴이 있긴 한데, 그것도 복잡함)
4. **조합의 위험** — 컨트랙트끼리 서로 호출 가능 → 예상치 못한 상호작용 발생

---

## 정리

- 이더리움 = 블록체인 + 프로그래밍
- 스마트 컨트랙트 = 블록체인 위의 자동 실행 프로그램
- EVM이 컨트랙트를 실행, 가스로 비용 제한
- 배포 후 수정 불가 → 보안이 생명
