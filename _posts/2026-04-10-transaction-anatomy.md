## 트랜잭션 = 블록체인의 HTTP 요청

Web2에서 서버에 요청을 보낼 때 HTTP 요청을 사용하듯이, Web3에서 블록체인에 무언가를 요청할 때는 **트랜잭션**을 전송합니다.

---

## 트랜잭션의 구조

```json
{
  "from":     "0xA...",        // 보내는 사람
  "to":       "0xB...",        // 받는 사람 (컨트랙트 주소일 수도 있음)
  "value":    "1000000000",    // 전송할 ETH (wei 단위)
  "data":     "0xa9059cbb...", // 호출할 함수 + 매개변수
  "gas":      21000,           // 최대 가스 한도
  "gasPrice": "20000000000",   // 가스당 가격
  "nonce":    42,              // 발신자의 트랜잭션 순번
  "chainId":  1                // 이더리움 메인넷
}
```

각 필드를 하나씩 살펴보겠습니다.

---

## from / to / value

- **from**: 발신자 주소. 개인키로 서명해야 유효합니다
- **to**: 수신자 주소. 일반 지갑일 수도, 스마트 컨트랙트일 수도 있습니다
- **value**: 전송할 ETH 양. 단위는 **wei** (1 ETH = 10^18 wei)

`to` 필드가 비어있는 경우 → **새 컨트랙트 배포**를 의미합니다.

---

## data (calldata)

**스마트 컨트랙트 함수를 호출**할 때 이 필드에 데이터가 담깁니다.

```
0xa9059cbb                                               ← 함수 셀렉터 (4바이트)
000000000000000000000000742d35cc6634c0532925a3b844bc9e75 ← 파라미터1: 주소
0000000000000000000000000000000000000000000000000de0b6b3a7640000 ← 파라미터2: 금액
```

- **함수 셀렉터**: `keccak256("transfer(address,uint256)")`의 앞 4바이트
- 나머지: ABI 인코딩된 매개변수

보안 관점에서 calldata는 중요합니다. **컨트랙트에 무엇을 지시하는지**가 여기에 모두 담겨 있기 때문입니다.

---

## Gas — 연료비

EVM에서 모든 연산에는 비용이 발생합니다.

| 연산 | 가스 비용 |
|------|-----------|
| 덧셈 | 3 |
| 곱셈 | 5 |
| 스토리지 쓰기 (신규) | 20,000 |
| 스토리지 쓰기 (수정) | 5,000 |
| ETH 전송 | 21,000 |

### 가스비 계산

```
실제 비용 = 사용된 가스 × 가스 가격(Gwei)

예: 50,000 gas × 30 Gwei = 1,500,000 Gwei = 0.0015 ETH
```

**가스가 부족하면?** → 트랜잭션이 실패합니다. 그런데 **이미 소비한 가스는 환불되지 않습니다.** 실패해도 비용은 청구됩니다.

---

## Nonce — 트랜잭션 순번

> 해당 주소에서 발송된 트랜잭션의 일련번호

```
첫 번째 트랜잭션: nonce = 0
두 번째 트랜잭션: nonce = 1
세 번째 트랜잭션: nonce = 2
```

왜 필요한가요?
- **재전송 공격(Replay Attack) 방지**: 같은 nonce의 트랜잭션은 한 번만 처리됩니다
- **순서 보장**: nonce 3이 처리되려면 0, 1, 2가 먼저 처리되어야 합니다

### 보안 이슈: nonce 갭

nonce 5를 전송하고 nonce 4를 전송하지 않으면 → nonce 5는 **대기 상태로 머뭅니다.** 이를 이용한 공격도 존재합니다.

---

## 트랜잭션 처리 흐름

```
1. 사용자가 트랜잭션을 생성합니다
2. 개인키로 서명합니다
3. 네트워크에 브로드캐스트합니다
4. 멤풀(Mempool)에서 대기합니다 ← 여기서 Front-running이 발생합니다!
5. 검증자가 블록에 포함합니다
6. EVM이 실행합니다
7. 상태 변경이 반영됩니다
8. 영수증(Receipt)이 발행됩니다
```

4번의 **멤풀**이 이후에 다룰 **Front-running / MEV** 공격의 무대입니다. 아직 처리되지 않은 트랜잭션이 공개적으로 노출되기 때문입니다.

---

## 정리

- 트랜잭션 = from, to, value, data, gas, nonce로 구성됩니다
- calldata = 컨트랙트에 내리는 명령
- 가스 = 연산 비용. 부족하면 실패하지만 비용은 그대로 청구됩니다
- nonce = 순번. 재전송 공격을 방지합니다
- 멤풀 = 대기실. 공격자가 내용을 엿볼 수 있는 공간입니다
