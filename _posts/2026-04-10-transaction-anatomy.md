## 트랜잭션 = 블록체인의 HTTP 요청

Web2에서 서버에 뭔가 하려면 HTTP 요청을 보내잖아? Web3에서 블록체인에 뭔가 하려면 **트랜잭션**을 보낸다.

---

## 트랜잭션의 구조

```json
{
  "from":     "0xA...",        // 보내는 사람
  "to":       "0xB...",        // 받는 사람 (컨트랙트 주소일 수도)
  "value":    "1000000000",    // 보내는 ETH (wei 단위)
  "data":     "0xa9059cbb...", // 호출할 함수 + 매개변수
  "gas":      21000,           // 최대 가스 한도
  "gasPrice": "20000000000",   // 가스당 가격
  "nonce":    42,              // 발신자의 트랜잭션 순번
  "chainId":  1                // 이더리움 메인넷
}
```

하나씩 뜯어보자.

---

## from / to / value

- **from**: 내 주소. 개인키로 서명해야 유효함
- **to**: 받을 주소. 일반 지갑일 수도, 스마트 컨트랙트일 수도 있음
- **value**: 보내는 ETH 양. 단위는 **wei** (1 ETH = 10^18 wei)

`to`가 비어있으면? → **새 컨트랙트 배포**라는 뜻이다.

---

## data (calldata)

**스마트 컨트랙트 함수를 호출**할 때 여기에 데이터가 들어간다.

```
0xa9059cbb                                               ← 함수 셀렉터 (4바이트)
000000000000000000000000742d35cc6634c0532925a3b844bc9e75 ← 파라미터1: 주소
0000000000000000000000000000000000000000000000000de0b6b3a7640000 ← 파라미터2: 금액
```

- **함수 셀렉터**: `keccak256("transfer(address,uint256)")`의 앞 4바이트
- 나머지: ABI 인코딩된 매개변수

보안 관점에서 calldata는 중요하다. **컨트랙트에 뭘 시키는지**가 여기 다 들어있으니까.

---

## Gas — 연료비

EVM에서 모든 연산에는 비용이 든다.

| 연산 | 가스 비용 |
|------|-----------|
| 덧셈 | 3 |
| 곱셈 | 5 |
| 스토리지 쓰기 (새로) | 20,000 |
| 스토리지 쓰기 (수정) | 5,000 |
| ETH 전송 | 21,000 |

### 가스비 계산

```
실제 비용 = 사용된 가스 × 가스 가격(Gwei)

예: 50,000 gas × 30 Gwei = 1,500,000 Gwei = 0.0015 ETH
```

**가스가 부족하면?** → 트랜잭션 실패. 그런데 **쓴 가스는 돌려받지 못한다.** 실패해도 돈은 나간다.

---

## Nonce — 순번

> 내 주소에서 보낸 트랜잭션의 일련번호

```
첫 번째 트랜잭션: nonce = 0
두 번째 트랜잭션: nonce = 1
세 번째 트랜잭션: nonce = 2
```

왜 필요한가?
- **재전송 공격(Replay Attack) 방지**: 같은 nonce의 트랜잭션은 한 번만 처리됨
- **순서 보장**: nonce 3이 처리되려면 0, 1, 2가 먼저 처리돼야 함

### 보안 이슈: nonce 갭

nonce 5를 보내놓고 nonce 4를 안 보내면? → nonce 5는 **펜딩 상태로 대기**. 이걸 이용한 공격도 존재한다.

---

## 트랜잭션 흐름 전체

```
1. 사용자가 트랜잭션 생성
2. 개인키로 서명
3. 네트워크에 브로드캐스트
4. 멤풀(Mempool)에 대기 ← 여기서 Front-running 발생!
5. 검증자가 블록에 포함
6. EVM이 실행
7. 상태 변경 반영
8. 영수증(Receipt) 발행
```

4번의 **멤풀**이 나중에 배울 **Front-running / MEV** 공격의 무대다. 아직 처리 안 된 트랜잭션이 공개적으로 보이기 때문.

---

## 정리

- 트랜잭션 = from, to, value, data, gas, nonce
- calldata = 컨트랙트에 "뭘 해라"는 명령어
- 가스 = 연산 비용. 부족하면 실패하지만 돈은 날아감
- nonce = 순번. 재전송 공격 방지
- 멤풀 = 대기실. 공격자가 엿볼 수 있는 곳
