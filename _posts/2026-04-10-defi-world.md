## DeFi = 은행 없는 금융

**DeFi (Decentralized Finance)** = 스마트 컨트랙트로 돌아가는 금융 서비스.

은행, 증권사, 보험사가 하는 일을 **코드**가 대신한다.

```
전통 금융                    DeFi
─────────────────────────────────────
은행 → 예금/대출           → Aave, Compound
증권거래소 → 주식 매매     → Uniswap, Curve
환전소 → 환전              → 1inch, SushiSwap
보험사 → 보험              → Nexus Mutual
```

---

## DEX — 탈중앙화 거래소

### 기존 거래소 (CEX: Centralized Exchange)

바이낸스, 업비트 같은 곳. **회사가 운영**하고, 네 자산을 **회사가 보관**한다.
- 장점: 빠르고, 사용하기 쉬움
- 단점: 해킹되면 끝 (FTX 사태: 80억 달러 증발)

### DEX (Decentralized Exchange)

**스마트 컨트랙트가 거래소**다. Uniswap이 대표적.

근데 문제가 있다. 기존 거래소는 **주문장(Order Book)**이 있잖아?

```
매수: ETH 3000달러에 10개 사겠다
매도: ETH 3100달러에 5개 팔겠다
```

블록체인에서 이걸 하려면 **매 주문마다 트랜잭션**을 보내야 한다. 가스비 폭탄.

그래서 나온 게 **AMM**이다.

---

## AMM — 자동 시장 메이커

> 주문장 없이 **수학 공식**으로 가격을 결정한다

### 핵심 공식: x × y = k

```
유동성 풀:
  ETH: 100개 (x)
  USDC: 300,000개 (y)
  k = 100 × 300,000 = 30,000,000

내가 1 ETH를 팔면:
  x = 101
  y = 30,000,000 / 101 = 297,029.7
  받는 USDC = 300,000 - 297,029.7 = 2,970.3 USDC
```

많이 바꿀수록 **가격이 불리해진다** (슬리피지). 이게 AMM의 핵심 특성이다.

### 유동성 공급자 (LP)

AMM에는 거래 상대가 없다. 대신 **유동성 풀**에 돈을 넣어둔 사람들이 있다.

```
유동성 공급자 → 풀에 ETH + USDC 넣음 → LP 토큰 받음
거래자 → 풀에서 스왑 → 수수료 발생 (보통 0.3%)
수수료 → 유동성 공급자에게 돌아감
```

## 렌딩 프로토콜 — 은행 대출

**Aave, Compound** 같은 프로토콜.

```
예금자: 1000 USDC 예치 → 이자 수익 (APY 3~10%)
대출자: ETH를 담보로 맡김 → USDC 대출 가능
```

### 과담보 대출

은행은 신용으로 대출해주지만, DeFi는 **신용이라는 개념이 없다.** (지갑 주소만 있으니까)

그래서 **담보 > 대출** 구조다.

```
ETH 2000달러어치 담보 → USDC 1400달러 대출 가능 (담보율 70%)
```

ETH 가격이 떨어져서 담보 가치가 대출 가치에 가까워지면? → **청산(Liquidation)**

```
담보 가치 ↓ → 건강 계수(Health Factor) 1.0 미만 → 누구든 청산 가능
청산자가 대출을 대신 갚고, 담보를 할인된 가격에 가져감
```

## DeFi의 보안 취약점 맛보기

DeFi는 **공격의 보물창고**다. 왜?

### 1. 조합 가능성 (Composability)

DeFi 프로토콜은 **레고**처럼 끼워맞춘다.

```
Uniswap에서 토큰 스왑 → Aave에서 대출 → Curve에서 유동성 공급
```

하나하나는 안전해도, **조합하면 예상 못한 취약점**이 생긴다.

### 2. Flash Loan

> 1 트랜잭션 안에서 **무담보로 수억 달러**를 빌릴 수 있다

같은 트랜잭션 안에서 갚기만 하면 된다. 이걸 이용한 공격은 13편에서 자세히.

### 3. Price Oracle 조작

AMM의 가격은 **풀의 비율**로 정해진다. 이 비율을 인위적으로 바꾸면? **가격 조작**.

**수천억 원**이 한 번의 트랜잭션으로 날아간다. 이게 Web3 보안의 현실이다.

## 정리

- DEX: 스마트 컨트랙트 기반 거래소 (Uniswap)
- AMM: x × y = k 공식으로 가격 결정
- 렌딩: 과담보 대출, 청산 메커니즘
- DeFi는 조합 가능 = 공격 표면도 조합 가능
- Flash Loan + Oracle 조작 = DeFi 해킹의 국룰 콤보
