## DeFi = 은행 없는 금융

**DeFi (Decentralized Finance)** = 스마트 컨트랙트로 구현된 금융 서비스입니다.

은행, 증권사, 보험사가 하는 일을 **코드**가 대신합니다.

```
전통 금융                    DeFi
─────────────────────────────────────
은행 → 예금/대출           → Aave, Compound
증권거래소 → 주식 매매     → Uniswap, Curve
환전소 → 환전              → 1inch, SushiSwap
보험사 → 보험              → Nexus Mutual
```

---

## DEX — 탈중앙화 거래소

### 기존 거래소 (CEX: Centralized Exchange)

바이낸스, 업비트 같은 곳입니다. **회사가 운영**하고, 사용자 자산을 **회사가 보관**합니다.
- 장점: 빠르고 사용하기 쉽습니다
- 단점: 해킹 시 자산이 위험합니다 (FTX 사태: 80억 달러 증발)

### DEX (Decentralized Exchange)

**스마트 컨트랙트 자체가 거래소**입니다. Uniswap이 대표적입니다.

그런데 기존 거래소에는 **주문장(Order Book)**이 있습니다:

```
매수: ETH 3000달러에 10개 구매
매도: ETH 3100달러에 5개 판매
```

블록체인에서 이를 구현하려면 **매 주문마다 트랜잭션**을 전송해야 합니다. 가스비 문제가 발생합니다.

이 문제를 해결하기 위해 **AMM**이 등장했습니다.

## AMM — 자동 시장 메이커

> 주문장 없이 **수학 공식**으로 가격을 결정합니다

### 핵심 공식: x × y = k

```
유동성 풀:
  ETH: 100개 (x)
  USDC: 300,000개 (y)
  k = 100 × 300,000 = 30,000,000

1 ETH를 판매하면:
  x = 101
  y = 30,000,000 / 101 = 297,029.7
  받는 USDC = 300,000 - 297,029.7 = 2,970.3 USDC
```

거래량이 많을수록 **가격이 불리해집니다** (슬리피지). 이것이 AMM의 핵심 특성입니다.

### 유동성 공급자 (LP)

AMM에는 거래 상대방이 없습니다. 대신 **유동성 풀**에 자금을 예치한 공급자들이 있습니다.

```
유동성 공급자 → 풀에 ETH + USDC 예치 → LP 토큰 수령
거래자 → 풀에서 스왑 → 수수료 발생 (보통 0.3%)
수수료 → 유동성 공급자에게 분배
```

## 렌딩 프로토콜 — 탈중앙화 대출

**Aave, Compound** 같은 프로토콜입니다.

```
예금자: 1000 USDC 예치 → 이자 수익 (APY 3~10%)
대출자: ETH를 담보로 예치 → USDC 대출 가능
```

### 과담보 대출

전통 금융은 신용으로 대출을 제공하지만, DeFi에는 **신용이라는 개념이 없습니다.** (지갑 주소만 존재하기 때문입니다)

따라서 **담보 > 대출** 구조입니다.

```
ETH 2000달러 담보 예치 → USDC 1400달러 대출 가능 (담보율 70%)
```

ETH 가격이 하락해 담보 가치가 대출 가치에 근접하면? → **청산(Liquidation)**

```
담보 가치 ↓ → 건강 계수(Health Factor) 1.0 미만 → 누구든 청산 가능
청산자가 대출을 대신 상환하고, 담보를 할인된 가격에 인수합니다
```

## DeFi의 보안 취약점 개요

DeFi는 **공격의 보물창고**입니다. 그 이유를 살펴보겠습니다.

### 1. 조합 가능성 (Composability)

DeFi 프로토콜은 **레고 블록**처럼 결합됩니다.

```
Uniswap에서 토큰 스왑 → Aave에서 대출 → Curve에서 유동성 공급
```

각각은 안전해도, **조합하면 예상치 못한 취약점**이 생길 수 있습니다.

### 2. Flash Loan

> 1 트랜잭션 안에서 **무담보로 수억 달러**를 빌릴 수 있습니다

같은 트랜잭션 내에서 상환하기만 하면 됩니다. 이를 이용한 공격은 #13편에서 자세히 다룹니다.

### 3. Price Oracle 조작

AMM의 가격은 **풀의 비율**로 결정됩니다. 이 비율을 인위적으로 변경하면? **가격 조작**입니다.

## 정리

- DEX: 스마트 컨트랙트 기반 거래소 (Uniswap)
